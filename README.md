## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Installation

```bash
$ yarn install
```

## Running the app

```bash
# development
$ yarn run start

# watch mode
$ yarn run start:dev

# production mode
$ yarn run start:prod
```

## Test

```bash
# unit tests
$ yarn run test

# e2e tests
$ yarn run test:e2e

# test coverage
$ yarn run test:cov
```

Your NestJS application should be running on http://localhost:3000.

Example Endpoints
Register a User:

POST /auth/register

json
Copy code
{
  "username": "user1",
  "password": "password"
}
Login a User:

POST /auth/login

json
Copy code
{
  "username": "user1",
  "password": "password"
}
Create a Task:

POST /tasks

Header: Authorization: Bearer {JWT_TOKEN}

json
Copy code
{
  "title": "New Task",
  "description": "Task Description"
}
Get All Tasks:

GET /tasks

Header: Authorization: Bearer {JWT_TOKEN}

Update a Task:

PUT /tasks/:id

Header: Authorization: Bearer {JWT_TOKEN}

json
Copy code
{
  "title": "Updated Task",
  "description": "Updated Description",
  "isCompleted": true
}
Delete a Task:

DELETE /tasks/:id

Header: Authorization: Bearer {JWT_TOKEN}

Documentation
Authentication:

JWT authentication is used.
Users can register and log in using their username and password.
Protected routes are secured with JwtAuthGuard.
CRUD Operations:

Users can create, retrieve, update, and delete tasks.
Each task has a title, description, completion status, and timestamp.
Environment Variables:

DB configuration and JWT secrets are stored in the .env file.
Error Handling:

Basic error handling is implemented, such as returning messages for invalid credentials.
Security:

Passwords are hashed using bcrypt before storing in the database.
JWT tokens are used to protect routes and ensure secure access.
This is a basic implementation. You can further extend the application with additional features like user roles, task assignments, notifications, etc.
